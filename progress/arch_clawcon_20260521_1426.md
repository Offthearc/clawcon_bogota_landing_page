# Architecture session — ClawCon Bogota Landing Page

Date: 2026-05-21 14:26 UTC

## Solution type agreed upon

**MVP** — first shippable static landing page, deployed to Vercel with no
server-side logic.

## Key architectural decisions

1. **Single-page, no routing.** All seven sections (Nav, Hero, About, Schedule,
   Guests, Registration CTA, Footer) compose into a single scrolling document.
   Anchor links (`#hero`, `#about`, etc.) handle in-page navigation.

2. **Static content only.** No `src/api/` layer needed. All event data is
   typed as constants co-located with their components (e.g.,
   `HeroSection/heroData.ts`). A future dynamic data source would introduce the
   `api/` layer at that point.

3. **One hook for hamburger state.** `useNavMenu` in `src/hooks/` owns the
   mobile menu open/close state and focus management. No external library.

4. **Mobile-first CSS** with breakpoints at 640 px (sm) and 1024 px (lg). Plain
   CSS Grid/Flexbox — no CSS framework introduced.

5. **Accessibility landmarks mandatory.** Each section has a correct HTML5
   semantic element, unique `id` anchor, `aria-labelledby` pointing to its
   `<h2>` heading, and an `aria-label` on the `<nav>`.

6. **Design tokens enforce no hardcoded values.** The `design:check` gate in
   `./init.sh` will reject any off-palette or off-token usage. The designer must
   update `DESIGN.md` with the bold/vibrant ClawCon palette before
   implementation begins.

7. **`scroll-behavior: smooth` gated on `prefers-reduced-motion`.** Applied only
   when the user has not requested reduced motion.

## Open questions / blockers

- `DESIGN.md` still contains the placeholder harness theme (neutral blues).
  The **designer agent must run before any implementer work** to set the
  ClawCon-specific palette (bold, vibrant, Latin American festival + gamer
  aesthetic). Implementation is blocked on this.
- Avatar images are all placeholders in the MVP; CSS-generated initials avatars
  are recommended so no real image assets are required.
- Registration CTA button href is a placeholder (`#register` or TBD). The real
  ticketing URL is out of scope for this MVP.

## Whether the implementer can proceed

**Not yet.** The designer must produce an updated `DESIGN.md` (and regenerate
`src/theme/tokens.css`) before implementation starts. Once `DESIGN.md` reflects
the ClawCon brand and `npm run design:check` passes, the implementer may begin
with F07 (NavBar) or F01 (Hero) as the first feature.
